Update your State Web Template to most current version

If your website is using a State Web Template that is perhaps couple of revisions back, you may be able to use the following method to update. Make extra backups if you plan to use the methods outlined below.

Before you start

Make a backup of your current site before you start. This way if you make any errors you still have your current site. Most template updates are made to make State Template more ADA compliant and for better compatibility in mobile devices.

Updating steps

  1. Download the latest release of the State Web Template release from our GitHub repository
  2. Copy over the core SASS files from /src/scss/ folder or compare the old and new SASS files. Compile/generate CSS from those updated SASS files and review your site.
    StateTemplate/
├── src/
  ├── scss/
  |   └── bootstrap/
  |   └── cagov/
  |   └── colorscheme/
  |   ├── cagov.core.scss
  |   └── variables.scss
  └── _includes/
  └── fonts/
  └── images/
  └── js/
  3. Copy over the js source files from /src/js/ folder or compare the old and new js files. Generate the output cagov.core.js file and run your website to test. 
    
├── src/
  ├── js/
  |   └── bootstrap/
  |   └── cagov/
  |   └── libs/
  |   └── index.js
  └── _includes/
  └── fonts/
  └── images/
  └── scss/
  4. Compare the structure of old and new page structure includes in /src/_includes/ folder. Here is the list of State Template webpage structure essential includes:
    
StateTemplate/
├── src/
    ├── _includes/
    |   └── layouts/
    |   └── modules/
    |   ├── analytics.njk
    |   ├── branding-lightgrey.njk
    |   ├── global-footer.njk
    |   ├── head-js-css.njk
    |   ├── mobile-controls.njk
    |   ├── navigation.njk
    |   ├── scripts.njk
    |   ├── search.njk
    |   ├── settings-bar.njk
    |   └── utility-header.njk
    └── _includes/
    └── fonts/
    └── images/
    └── scss/
    Make sure your code patters and structure matches the new includes code.
  5. Copy over the fonts files from /src/fonts/ folder
  6. Compare structure and code patterns of your State Tempalte /src/_includes/modules/ that you are using in your website with new components/modules structures. Update your code if nessesary.
  7. Test all your pages.

Use State Template CDN

Update your website faster by using State Template CDN. You will only need to update CSS and JavaScript CDN links. No manual copying over files is needed. Instead, you can replace copying over files described in steps 2 and 3 with CDN. By doing so you will eliminate step 5 because fonts are already referenced in CDN import calls.
