Getting Started with California State Template

The following page includes examples of applying California State Template css, js and includes files to a web page. Here are the steps you need to quickly set you your site with California State Template.

Note: Load your website faster by using State Template CDN

Contents

To get started you need to download Californis State Template from GitHub. Once downloaded, unzip the compressed folder and you’ll see something like this:

/State Web Template/
├── /_site/
└── /pages/
  ├── index.html
  └── /sample/
    ├── accordions.html
    ├── color-schemes.html
    └── etc...
└── /src/
  └── /_includes/
    ├── global-footer.njk
    └── /layouts/
      ├── base.njc
      └── /modules/
       ├── accordion.html
       └── etc...
  └── /fonts/
  └── /images/
  └── /js/
  └── /scss/
├── eleventy.js
├── package.json
└── rollup.config.js
  • _site – This folder contains website's output files.
  • pages – Contains website's input pages that a developer or e content editor can edit.
  • src – The “src” folder contains includes, modules, fonts, images, SCSS and JavaScript files.
  • images – This folder contains all the images used by the State Template.
  • js – Contains JavaScript files that are used by the State Template. Third party JavaScript libraries are also included in this folder.
  • _includes – This folder contains the common template components used on various pages in the State Template (for example, header, footers, navigation, etc.).

For more information about State Template's structure go to our Structure page.

Running State Web Template locally

Download Node.js

Make sure your local machine has Node installed. To see if Node is installed, open the Windows Command Prompt, Powershell or a similar command line tool, and type:

node -v

This should print the version number. Othervise you can download it from nodejs.org

Using npm to start the website locally

Once Node is installed, and once you are in your State Web Template local directory run following npm commands to build and run the website locally:

npm install
This command will install all the nessesary npm packages to your local website directory. 
npm run build
This command will build and generate website's pages, css and js files. 
npm run start
This command will run the website in your http://localhost:8080/ and watch for any updates you are making to webpages or scss and js files.

Output

All the web content is generated into /_site/ folder, which is website's final output folder.

CSS

State Web Templagte core css files are generated from SASS files that can be found in /src/scss/ folder. Multiple SASS files are compiled during the build process into core output CSS files that are generated in /_site/css/ folder. A State Web Template webpage requires these two CSS files:

<link rel="stylesheet" href="/css/cagov.core.css">
<link rel="stylesheet" href="/css/colorscheme-oceanside.css">

If your website doesn't use a site generator you can simply copy-paste the stylesheet links into your <head> before all other stylesheets to load our CSS. please note that you can replace colorscheme-oceanside.css with color scheme file of your choice.

JS

A State Web Template webpage requires cagov.core.js JavaScript file to function. That file is generated by rollup.js processor from multiple JavaScript files that can be found in /src/js/ folder. If your website doesn't use a site generator place the following near the end of your pages, right before the closing </body> tag. 

<script src="/js/cagov.core.js"></script>

Note: Starting from release 5.6.0 State Template stopped using any jQuery libraries, or any components that are using jQuery.

Clean Starter Template

Be sure to have your pages set up with the latest design and development standards. That means using an HTML5 doctype and including a viewport meta tag for proper responsive behaviors. Put it all together and your pages should look like this: 

<!doctype html>
<html lang="en">
  <head>
    <!-- Required meta tags -->
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">

    <!-- State Template CSS -->
    <link rel="stylesheet" href="/css/cagov.core.css">
    <link rel="stylesheet" href="/css/colorscheme-oceanside.css">

    <title>California State Webpage</title>

  </head>
  <body>
    <h1>California State Webpage</h1>

    <!-- State Template JavaScript -->
    <script src="/js/cagov.core.js" ></script>
  </body>
</html>

Note: Template above does not include any State Template content, modules or server side includes.

Using Includes

Another way to set up the Californisa State Template web page is by using eleventy includes. Since 6.0 State Web Template is using eleventy framework to build and generate webpages the most essential components such as page headers, footers, navigation are already included in our webpage structure. Here is code example of our base.njk layout page that is using essential njk includes: 

<!doctype html>
<html lang="en">
   <head>
      <meta charset="utf-8">
      <title>State Template</title>
      <meta name="Author" content="State of California" />
      <meta name="Description" content="State of California" />
      <meta name="Keywords" content="California, government" />
      ❴% include "/src/_includes/head-css-js.njk" %❵
   </head>
   <body>
      <header role="banner" id="header" class="global-header fixed">
         <div id="skip-to-content"><a href="#main-content">Skip to Main Content</a></div>
         ❴% include "/src/_includes/utility-header.njk" %❵/span>
         ❴% include "/src/_includes/settings-bar.njk" %❵
         ❴% include "/src/_includes/branding-lightgrey.njk" %❵
         ❴% include "/src/_includes/mobile-controls.njk" %❵
         <div class="navigation-search full-width-nav container">
            <div id="head-search" class="search-container search-container featured-search">
              ❴% include "/src/_includes/search.njk" %❵
            </div>
            ❴% include "/src/_includes/navigation.njk" %❵
         </div>
      </header>
      <div id="main-content" class="main-content">
         <div class="container">
              <main class="main-primary">
		❴{ content | safe }}
              </main>
         </div>
      </div>
      ❴% include "/src/_includes/global-footer.njk" %❵
      ❴% include "/src/_includes/scripts.njk" %❵
   </body>
</html>

Note: Path to the includes will be relative to the directory of your base.njk page.
